健康检查脚本会检查以下项目：

| 检查项目 | 说明 |
|---------|------|

| 必需字段 | 检查 `type`、`site`、`steps` 字段 |
| 步骤结构 | 验证 `steps` 是否为非空列表 |
| 视频类型 | 验证视频类型是否有效 |
| 配置字段 | 检查 `config` 字段（如果有）的结构 |

## 🚀 使用方法

### 方式 1：Makefile（推荐）
```bash
make check-health
```

### 方式 2：直接运行脚本
```bash
# 基本检查
python scripts/check_health.py

# 指定刮削源目录
python scripts/check_health.py --dir /path/to/scrapeflows

# 保存报告到文件
python scripts/check_health.py --save health_report.json
```

### 方式 3：集成到工作流程
```bash
# 打包前先检查
make check-health && make package
```

## 🛠️ 最佳实践建议

### 1. 定期检查
建议在以下情况下运行健康检查：
- 新增或修改刮削源后
- 打包发布前
- 定期（如每周一次）检查维护

### 2. Git 钩子
可以设置 git 钩子，在提交前自动检查：
```bash
# .git/hooks/pre-commit
make check-health
if [ $? -ne 0 ]; then
  echo "刮削源有问题，提交取消！"
  exit 1
fi
```

### 3. CI/CD 集成
如果使用 CI/CD 系统，可以在 pipeline 中集成：
```yaml
# .github/workflows/ci.yml
name: CI
on: [push, pull_request]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: make check-health
      - run: make test
```
